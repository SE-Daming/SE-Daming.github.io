OOP 思想总结

在这个项目中，面向对象编程（OOP）的思想得到了充分体现。以下是项目中主要使用的 OOP 概念和设计思想的总结：

1. 封装（Encapsulation）

  • 数据隐藏：通过将类的属性设置为 private，并通过 public 方法（如 getter 和 setter）访问和修改这些属性，实现了数据的封装。例如：
    • Mario 类中的 score 属性是私有的，通过 getScore() 和 addScore() 方法进行访问和修改。
    • Hammer 类中的 collected 属性是私有的，通过 isCollected() 和 collect() 方法进行访问和修改。
  • 行为封装：将相关的操作封装在类的方法中，例如 Mario 类中的 update() 方法封装了 Mario 的移动、跳跃和攀爬逻辑。

2. 继承（Inheritance）

  • 基类与子类：项目中使用了继承来复用代码。例如：
    • Entity 类作为基类，定义了所有游戏实体的通用属性和方法（如 position、image、render() 等）。
    • Mario、DonkeyKong、Hammer、Barrel、Ladder 和 Platform 等类继承自 Entity，并扩展了各自的特有行为。
  • 代码复用：通过继承，子类可以直接使用基类的属性和方法，减少了重复代码。

3. 多态（Polymorphism）

  • 方法重写：子类可以重写基类的方法以实现特定行为。例如：
    • Mario 类重写了 Entity 类的 update() 方法，实现了 Mario 特有的移动和跳跃逻辑。
    • Hammer 类重写了 Entity 类的 update() 方法，使其保持静止。
  • 统一接口：通过基类 Entity 的 render() 和 update() 方法，可以统一处理所有游戏实体的渲染和更新逻辑。

4. 抽象（Abstraction）

  • 抽象类：Entity 类是一个抽象类，定义了游戏实体的通用行为（如 render() 和 update()），但具体的实现由子类完成。
  • 简化复杂性：通过抽象，将复杂的游戏实体行为分解为更小的、可管理的部分。

5. 组合（Composition）

  • 对象组合：ShadowDonkeyKong 类通过组合的方式管理多个游戏实体（如 Mario、DonkeyKong、Hammer 等），而不是通过继承。
  • 灵活性：通过组合，可以动态地添加或移除游戏实体，增强了代码的灵活性。

6. 单一职责原则（Single Responsibility Principle）

  • 职责分离：每个类只负责一个特定的功能。例如：
    • Mario 类负责 Mario 的移动、跳跃和攀爬逻辑。
    • Hammer 类负责锤子的状态管理。
    • ShadowDonkeyKong 类负责游戏的整体逻辑和状态管理。

7. 开闭原则（Open/Closed Principle）

  • 扩展开放，修改关闭：通过继承和多态，可以在不修改现有代码的情况下扩展功能。例如：
    • 如果需要添加新的游戏实体（如 Fireball），只需创建一个新的类继承自 Entity，并实现其特有行为，而不需要修改 ShadowDonkeyKong 类。

8. 依赖倒置原则（Dependency Inversion Principle）

  • 高层模块不依赖低层模块：ShadowDonkeyKong 类依赖于抽象的 Entity 类，而不是具体的子类（如 Mario 或 Hammer），这使得代码更易于维护和扩展。

9. 状态模式（State Pattern）

  • 游戏状态管理：通过 GameState 枚举类管理游戏的不同状态（如 HOME、PLAYING、GAME_OVER），并在 update() 方法中根据当前状态执行不同的逻辑。

总结

这个项目充分运用了 OOP 的核心思想，通过封装、继承、多态、抽象、组合等特性，构建了一个结构清晰、易于扩展和维护的游戏框架。OOP 的设计原则（如单一职责、开闭原则、依赖倒置等）也得到了很好的体现，使得代码具有较高的可读性和可维护性。

为什么用OOP？

  1. 代码复用：继承和组合减少重复代码
  2. 模块化：像搭积木一样构建复杂系统
  3. 易于维护：修改一个对象不影响其他部分
  4. 贴近现实：用对象思维建模更直观

Bagel API 使用总结

Bagel 是什么？

Bagel 是一个基于 Java 的轻量级 2D 游戏引擎，专为教学和快速开发简单游戏而设计。它提供了易于使用的 API，帮助开发者快速实现游戏的核心功能，如窗口管理、图形渲染、输入处理和游戏循环。

核心类

  • AbstractGame: 游戏主类继承的基类，提供游戏循环和更新机制
  • Window: 管理游戏窗口，提供窗口尺寸和关闭功能
  • Input: 处理键盘和鼠标输入

图形渲染

  • Image: 用于加载和绘制游戏中的图片资源
  • Font: 用于绘制文本，支持设置字体和字号
  • Point: 表示2D空间中的坐标点
  • Rectangle: 用于表示矩形区域，主要用于碰撞检测

游戏循环

  • update(): 每帧调用的方法，处理游戏逻辑和渲染
  • run(): 启动游戏主循环

输入处理

  • Keys: 枚举类，表示键盘按键
  • isDown(): 检测按键是否按下
  • wasPressed(): 检测按键是否刚被按下

实用工具

  • getWidth()/getHeight(): 获取窗口尺寸
  • getBoundingBoxAt(): 获取图片的边界框
  • intersects(): 检测两个矩形是否相交
