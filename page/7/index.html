bean的生命周期

2024年10月27日 留言

1

大致分为四步

实例化=》属性赋值=》初始化=》使用、销毁

创建 Bean 的实例：Bean 容器首先会找到配置文件中的 Bean 定义，然后使用 Java 反射 API 来创建 Bean 的实例。

Bean 属性赋值/填充：为 Bean 设置相关属性和依赖，例如@Autowired 等注解注入的对象、@Value 注入的值、setter方法或构造函数注入依赖和值、@Resource注入的各种资源。

Bean 初始化：

  • 如果 Bean 实现了 BeanNameAware 接口，调用 setBeanName()方法，传入 Bean 的名字。
  • 如果 Bean 实现了 BeanClassLoaderAware 接口，调用 setBeanClassLoader()方法，传入 ClassLoader对象的实例。
  • 如果 Bean 实现了 BeanFactoryAware 接口，调用 setBeanFactory()方法，传入 BeanFactory对象的实例。
  • 与上面的类似，如果实现了其他 *.Aware接口，就调用相应的方法。
  • 如果有和加载这个 Bean 的 Spring 容器相关的 BeanPostProcessor 对象，执行postProcessBeforeInitialization() 方法
  • 如果 Bean 实现了InitializingBean接口，执行afterPropertiesSet()方法。
  • 如果 Bean 在配置文件中的定义包含 init-method 属性，执行指定的方法。
  • 如果有和加载这个 Bean 的 Spring 容器相关的 BeanPostProcessor 对象，执行postProcessAfterInitialization() 方法

销毁 Bean：销毁并不是说要立马把 Bean 给销毁掉，而是把 Bean 的销毁方法先记录下来，将来需要销毁 Bean 或者销毁容器的时候，就调用这些方法去释放 Bean 所持有的资源。

  • 如果 Bean 实现了 DisposableBean 接口，执行 destroy() 方法。
  • 如果 Bean 在配置文件中的定义包含 destroy-method 属性，执行指定的 Bean 销毁方法。或者，也可以直接通过@PreDestroy 注解标记 Bean 销毁之前执行的方法。

AQS是什么

2024年10月27日 留言

定义

AQS是Java中用户构建锁和同步器的一个抽象工具类，它的核心思想是如果被请求的资源空闲则将请求线程设为有效的工作线程，将资源设为锁定状态。如果资源被占用则需要一定的阻塞等待机制来保证锁分配。它的核心有三部分：状态state、控制线程获得锁的队列、实现类要实现的方法（tryAcquire\tryRelease）.比如常见的锁和同步器ReentrantLock、Semaphore、countdownLatch等都是基于AQS构建的

MVCC和最左前缀法则

2024年10月27日 留言

1

MVCC

定义

MVCC是数据库中的多版本并发控制机制、通过维护数据的多个历史版本实现并发事务同时读写数据库时保持数据的一致性和隔离性。它允许多个事务同时读取同一行数据而不会相互阻塞，每个事务看到的数据版本是事务刚开始时的数据版本

实现

undolog、readview、隐藏字段（最后一次插入或更新该数据事务id、回滚指针）

能解决的问题

能解决RR级别下的不可重复读和部分幻读

RC下事务中每次查询都生成readview、RR下只在事务开始前生成一次，因此可重复读

最左前缀法则

在查询数据时根据联合索引的创建顺序从左到右依次匹配查询条件的字段。遇到范围查询时（> <）该字段索引及其后面的索引就会失效

RBAC模型介绍一下

2024年10月26日 留言

它是通过给角色分配权限、给用户分配角色来实现权限分配的、在数据库涉及到5张表—用户表、权限表、角色表、用户角色表、角色权限表

CMS平台有不同的角色、管理员、编辑、审核等等；权限有创建内容、编辑、审核、发布、IP控制、黑名单等等

优点

便于角色的划分、授权管理、赋予最小权限、职责分离

缺点

没有提供按顺序操作的控制机制、很难适应对操作顺序要求严格的系统

如何解决缺点

基于ABAC模型

ABAC是一种基于属性的权限控制模型，它通过用户（用户组织、姓名）、资源（资源所有者、文件名）和环境（访问时间、数据的位置）的属性来决定访问权限。与RBAC不同，ABAC不依赖于预定义的角色，而是动态评估属性，以做出访问决策。

ABAC可根据用户特征、操作类型等属性来进行授权、RBAC是根据角色授权

假设有一个文件系统，用户A是财务部门的员工，文件X是敏感文件。你可以设置以下策略：

  • 只有财务部门的员工（用户属性）才能访问（文件属性）。

  • 只有在工作时间（环境属性）内才能访问。

P5（职级）的研发（职位）同学有公司Gitlab的权限

优点

更加灵活、可扩展

缺点

构建比较复杂

SpringBoot自动装配原理

2024年10月25日 留言

自动装配原理

springboot通过SPI机制定义了一套接口规范、规定在启动时会扫描META-INF/spring.factories文件、将文件中的配置信息加载到spring容器

具体实现与@SpringBootApplication注解有关、该注解下有个@EnableAutoConfiguration注解、该注解内加载了AutoConfigurationImportSeletor类，这个类中的selectImports方法加载自动装配类到IOC容器。该方法中有三个主要步骤、

先看自动装配是否打开，然后获取EnableAutoConfiguration注解内的exclude和excludeName有哪些、然后根据condition注解获取需要自动装配的配置类、读取META/spring factories里的实现类

如何自定义starter

创建springboot工程、导入springboot依赖、创建一个配置类，在META/spring factories下写入该类的全类名、导入该工程、完成
